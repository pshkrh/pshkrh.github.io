Time to welcome a new year.

## 2019 Is Here!

Every New Year brings about a New Year's Resolution, and this year is no different.

This year I'd like to accomplish a few tasks, instead of a single goal.

* Learn to Code, specifically Python
* Learn the actual best coding practices
* Learn Machine Learning, Deep Learning and get some projects / models deployed

This list looks small, but these tasks will take quite a bit of time. I've already started the fast.ai v3 course however, so I'm well on the way for completing these 3 resolutions.

I'll be trying my best to post a weekly update and / or my own notes for each week of fast.ai as I go through the course. Post that, I'll be striving to write some blog posts about projects I will be implementing using the knowledge I get from the course.

I'm also planning on pursuing the Deep Learning specialization on Coursera (by Andrew Ng). I've heard good things about it, and I'm excited to learn more in the future.

The hardest part of the above resolutions though, for me at least, is the coding. I've never really been good at coding (by coding I mean competitive programming type of problems, where the main focus is logic), mostly because I've never tried hard. Learning to code isn't easy, and there's no simple way around it. I always reach a certain point after which I am no longer able to solve problems because I just don't know "what to do".

After researching a bit on this block, some folks over on programming subreddits just say to look at other people's solutions. I've always thought looking at other's solutions was very cheaty, but after thinking on it a while, it makes complete sense.

There's always going to be someone smarter than you, and if you're stuck then look at the smart person's solution, understand the reasoning behind said solution, then write your own.

This way, I think I'll be able to get past the "coding block" as I like to call it, and get better. Libraries like NumPy and Pandas are essential for Data Science and it's necessary that a workman know his tools well and how they work in order to make his final product good.

That's all for now.

Ciao!
